## 搬瓦工控制面板是什么？

搬瓦工控制面板的正式名称是 **KiwiVM Control Panel**，是搬瓦工官方自己开发的 VPS 管理后台。买了搬瓦工的每台 VPS，都会有一个独立的 KiwiVM 面板与之对应。

你能在这里做的事情包括：查看 VPS 的 IP 地址和 SSH 端口、开关机和重启、重装操作系统、迁移机房、创建快照备份、修改 root 密码，以及查看流量使用情况。可以理解成这台服务器的"驾驶舱"，绝大多数日常管理操作都在这里完成。

KiwiVM 控制面板的直接访问地址是：

> **kiwivm.64clouds.com**

书签收藏这个地址就行，以后要直接登录面板直接打开这个链接。

不过正常情况下你不需要手动输这个地址——通过搬瓦工官网客户区跳转进去更方便，下面说两种方式。

## 两种登录方法

### 方法一：从官网客户区跳转（推荐）

这是最常用的方式，不需要记密码，点几下就进去了。

1. 打开搬瓦工官网，点右上角的 **Client Area** 登录账号
2. 顶部菜单找到 **Services → My Services**
3. 列表里点你要管理的那台 VPS
4. 进入详情页后，找到 **KiwiVM Control Panel** 按钮，点击跳转

系统会自动完成认证，直接进入这台 VPS 的面板主界面。全程不需要输入 VPS 的 IP 或任何额外密码。

用这种方式登录比较安全，日常管理基本靠这个。

---

### 方法二：直接访问面板地址

偶尔你可能需要把 KiwiVM 的访问权限单独给别人（比如把服务器借给朋友管），这时候方法一不行——总不能把账号密码也一起给对方。这种情况用直接登录的方式：

1. 打开 `kiwivm.64clouds.com`
2. 在 **VPS IP address** 栏填你服务器的 IP
3. 在 **KiwiVM password** 栏填 KiwiVM 面板的专属密码（**注意：这个密码跟搬瓦工账号密码、服务器 root 密码都不一样，是独立的**）
4. 点 Log in

第一次用的朋友可能不知道 KiwiVM 密码是什么——默认情况下，搬瓦工不会单独发这个密码，你得先通过方法一进入面板，然后在左侧菜单找到 **KiwiVM password modification** 自己设一个。设完之后才能用方法二登录。

## KiwiVM 面板主要功能介绍

进了面板，左边是导航菜单，右边是当前 VPS 的状态概览。下面把常用功能逐一说一遍。

### Main controls（主控制台）

这是进入面板看到的第一个页面，显示的是 VPS 的基础信息：

- 当前运行状态（Running / Stopped）
- IP 地址和 SSH 端口
- 内存、硬盘占用情况
- 本月流量使用进度

操作按钮有四个：**Start（开机）、Stop（关机）、Reboot（重启）、Force Stop（强制关机）**。机器卡死的时候用 Force Stop，正常情况用 Stop 就行。

---

### Install new OS（重装系统）

搬瓦工支持 CentOS、Debian、Ubuntu 等多个 Linux 发行版，可以在这里一键切换。

操作步骤：

1. 先回到 Main controls 把 VPS **Stop**（必须先关机才能重装）
2. 进入 Install new OS
3. 在下拉列表选操作系统版本
4. 勾选确认（会清空所有数据的提示）
5. 点 Reload

重装速度挺快，通常 1-2 分钟完成。完成后页面会显示新的 **root 密码** 和 **SSH 端口**，这两个信息只显示一次，必须马上复制保存好，关掉页面就找不回来了。

---

### Root shell（在线终端）

KiwiVM 提供了三种在线终端工具：

- **Basic**：基础命令行，功能有限
- **Advanced**：功能更完整
- **Interactive**：交互式终端，体验最接近正常 SSH

SSH 连不上的时候，可以用 Interactive 直接进系统操作——比如换 SSH 端口、重置密码、排查服务问题，都能用这个救急。

---

### Migrate to another DC（迁移机房）

支持多机房切换的套餐（比如 CN2 GIA-E）可以在这里免费迁移机房，次数不限。

迁移后 IP 会变，数据会保留（但迁移过程会消耗一定流量）。操作很简单，选目标机房，点确认，等系统完成迁移就行。

---

### Snapshots & Backups（快照与自动备份）

- **Snapshots（快照）**：手动创建整台 VPS 的完整备份。快照默认只保存 30 天，如果想永久保留可以设置成 Sticky 状态。
- **Backups（自动备份）**：搬瓦工提供免费的自动备份功能，每隔一段时间自动给 VPS 做一次完整备份，出了问题可以一键恢复。

数据备份这事能做就做，不然哪天不小心重装系统忘记备份，就真什么都没了。

---

### Root password modification / KiwiVM password modification

前者重置服务器的 root 密码，后者修改 KiwiVM 面板的独立登录密码。两个密码互不干扰。忘了 root 密码点 Generate 直接生成新的，马上在页面里显示，同样要及时复制保存。

---

### Two-factor authentication（两步验证）

开启后每次登录 KiwiVM 面板需要额外输入一个验证码，安全性明显提升。如果这台服务器上有重要数据，建议开一下。

---

### API

显示这台 VPS 的 API Key 和 VEID，需要用代码管理服务器的可以在这里取。

## 常见问题处理

### KiwiVM 报错 Session Expired

这是用得最多的朋友反馈最多的问题。进了面板一直提示：

> *Your session has timed out due to inactivity. Please log in again to continue.*

直接重新点 KiwiVM Control Panel 进去也还是这个报错。

**原因**：搬瓦工在某次 KiwiVM 面板更新后加强了 Cookie 安全机制，旧 Cookie 和新版本冲突了。

**解决方法**（两选一）：

1. **快速临时解决**：开一个无痕浏览器窗口，用方法一重新进入面板，不会受旧 Cookie 影响。

2. **彻底解决**：清除浏览器里 `64clouds.com` 和 `kiwivm.64clouds.com` 的所有 Cookie。Chrome 下按 F12 打开调试工具，切换到"应用 → Cookies → 删除所有"。清完后重新登录就正常了。

---

### 面板打开超时 / kiwivm.64clouds.com 连接不上

偶尔搬瓦工会有短暂的服务维护或者网络抖动，这时候直接访问 kiwivm.64clouds.com 会超时。

几个排查步骤：

1. 先等几分钟再试，大多数情况是暂时性的
2. 换个网络环境（比如手机热点）重新试一次
3. 清除浏览器缓存后重试

如果等了十几分钟还连不上，可以通过搬瓦工官网客服提交工单，看是否是机房层面的故障。

---

### 面板找不到 SSH 端口

登录 KiwiVM 后在 Main controls 页面里应该能直接看到 SSH Port。

找不到通常有两种情况：

- **VPS 没有安装系统**：需要先去 Install new OS 安装一个操作系统。
- **系统出了问题**：尝试在 Main controls 里关机再开机（Stop → Start），等两分钟再刷新页面。

## 常见问题 FAQ

**Q：搬瓦工 KiwiVM 控制面板的密码是什么？**
A：KiwiVM 有自己独立的面板密码，和账号密码、服务器 root 密码完全是三个不同的东西。默认不强制设置，第一次用方法一（官网跳转）进入面板后，在左侧 KiwiVM password modification 里自己设一个就行。

**Q：KiwiVM 打开后一直是 Session Expired 怎么办？**
A：清除浏览器中 `64clouds.com` 的 Cookie，或者直接开无痕窗口重新进入。这是搬瓦工某次安全更新留下的兼容问题，清完 Cookie 就恢复正常了。

**Q：在 KiwiVM 里迁移机房会丢数据吗？**
A：数据会保留，IP 地址会变更。迁移过程中 VPS 会短暂不可访问，迁移完成后用新 IP 重新连接即可。建议迁移前先创建一个快照备份以防万一。

**Q：KiwiVM 面板里的 Root shell 和 SSH 有什么区别？**
A：效果基本一样，都是操作服务器命令行。区别在于 Root shell 是通过搬瓦工的面板页面操作，不需要 SSH 客户端，在 SSH 连不上的时候可以用来救急。

**Q：搬瓦工不同套餐的 KiwiVM 面板功能有区别吗？**
A：大部分功能是一样的，但机房迁移这个功能只对支持多机房切换的套餐有效（比如 CN2 GIA-E）。香港套餐和一些限量版套餐最初锁定单一机房，部分后来已开放切换权限。
